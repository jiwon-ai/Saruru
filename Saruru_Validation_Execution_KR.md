# 사르르 — 검증 실행 패키지 (턴키)

> 검증 키트(`Saruru_Validation_Kit.md`)·스크립트(`Saruru_Validation_Scripts.md`)와 함께. 이거 그대로 따라 하면 **1주차 검증이 굴러간다.**

## 0. 이번 주 체크리스트 (Day-by-day)
- **D1:** 랜딩 배포 + Tally 폼 2개(개인/사장) + 추적 시트 열기 + 위저드오즈 1~2명
- **D2:** 지인 사장님 매장에서 3~5명 + **상표검색(KIPRIS)**
- **D3:** 영상 1편 촬영·업로드 → 랜딩 트래픽
- **D4–5:** 위저드오즈 누적 10~15명, DM 모집
- **D6–7:** 영상 2편, 누적 20명 목표, 중간 점검

## 1. 랜딩 배포 (5분)
- 가장 쉬움: **Netlify Drop**(app.netlify.com/drop)에 `Saruru_FakeDoor_Landing.html`을 드래그 → 즉시 공개 URL.
- 또는 **GitHub Pages**: repo Settings → Pages → Branch: main /(root) → 저장.
- 배포 전, 랜딩의 두 버튼 `href`(REPLACE_FORM_URL_*)를 아래 Tally 폼 주소로 교체.

## 2. 사전등록 폼 (Tally, 무료, 10분)
- tally.so → 폼 **2개**: "사르르 — 개인" / "사르르 — 사장님".
- 질문: **이메일(필수)** · 한 줄 "왜 관심 있어요?" · (사장폼) 직원 수.
- **폼 응답 수 = 지불의향 신호.** 개인 vs 사장 응답 비교 = 구매자 검증(Risk B).

## 3. 모집 DM 템플릿 (복붙용)
**(a) 지인 사장님**
> "사장님, 저 요즘 서비스직 분들 감정 케어 앱을 준비 중이에요. 손님 때문에 힘들었던 일 하나만 들려주시면, 제가 마음 정리되는 한마디 보내드릴게요. 3분이면 돼요. 도와주실 수 있을까요? (커피 기프티콘 드려요 ☕)"

**(b) 카페알바·자영업 커뮤니티 글**
> "진상 손님 때문에 퇴근하고도 곱씹는 분 계세요? 그 감정 정리하는 법을 테스트 중이에요. 사건 하나 익명으로 들려주시면 정리 멘트 보내드려요(3분, 기프티콘). 댓글/DM 주세요."

**(c) 인스타 DM**
> "안녕하세요! 감정노동 케어 앱을 만들고 있어요. 손님 때문에 힘들었던 일 하나만 들려주실 수 있을까요? 마음 정리되는 한마디 보내드릴게요. 3분, 커피 한 잔 보답드려요 :)"

## 4. 위저드오즈 진행 (10~15분/명)
1. 사건 하나 듣기 → `Saruru_Validation_Scripts.md`의 리프레임 템플릿으로 3~5문장 + 녹임 문장 전달(카톡).
2. 직후 4질문(효능 0~10 / 재방문 / 빈도 / 열린질문) → 추적 시트 기록.
3. **밤 의식 질문 추가:** "이런 거, **자기 전에 하루 정리로 매일** 쓸 것 같아요?" → 밤 의식 가설(데일리 트리거) 검증.

## 5. 추적 시트
- 동봉한 **`Saruru_Tracking.xlsx`**에 한 명당 한 줄 기록. (없으면 구글시트로 같은 컬럼 복제.)
- '요약' 시트에 효능 평균·재방문 비율·밤의식 비율이 자동 계산됨.

## 6. 상표검색 (D2, 30분)
- **KIPRIS**(kipris.or.kr): "사르르", "Saruru" → **9류·42류** live 상표 확인.
- **WIPO**(branddb.wipo.int): "Saruru".
- 9·42류에 동일/유사 없으면 진행 OK. (정식 출원 전 변호사 1회 권장.)

## 7. 판정 (4주차)
- 검증 키트 §5 **kill criteria**로 통과 / 피벗 / 중단 결정.
- 추가 신호: 밤 의식("매일 쓸 것 같다") 비율이 높으면 → 데일리 습관·구독 모델 청신호(Risk A 완화).
